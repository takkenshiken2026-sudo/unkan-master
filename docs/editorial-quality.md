# 編集品質基準（専門家 × プロライター）

試験ガイド（`guide_articles.csv`）と用語詳細（`glossary_terms.csv`）を、**資格の専門家が監修し、プロのライターが執筆した記事**と同程度の読み味・信頼性・具体性で公開するための基準です。

- 公開境界・SEO骨格: [seo-article-guidelines.md](./seo-article-guidelines.md)
- 試験ガイド CSV: [guide-article-template.md](./guide-article-template.md)
- 用語詳細 CSV: [glossary-term-template.md](./glossary-term-template.md)
- 機械判定: `tools/editorial_quality.py`（共通） / `tools/guide_article_rules.py` / `tools/glossary_term_rules.py`
- **試験ガイドと用語解説の立ち位置:** [content-positioning.md](./content-positioning.md)

---

## コンテンツの役割分担（試験ガイド vs 用語解説）

| | 試験ガイド | 用語詳細・知識ハブ |
|---|------------|-------------------|
| **問い** | どう進めるか | 〇〇とは／違いは |
| **単位** | 1検索意図（計画・手続） | 1語・1比較・1数値セット |
| **書くこと** | 手順・計画・演習の回し方 | 定義・試験論点・誤解 |
| **書かないこと** | 用語の定義を主題にしない | 学習スケジュール・申込手順 |

詳細な振り分け表・内部リンクルールは [content-positioning.md](./content-positioning.md) を参照。

---

## プロ記事の定義（このテンプレでの到達点）

読者が記事を読んだあと、次の3つが満たされている状態を目指します。

1. **判断できる** … 何を公式で確認し、何を演習で確かめればよいかが分かる
2. **試験に結びつく** … 定義だけでなく、選択肢で落とし穴になる点が書かれている
3. **信頼できる** … 執筆・確認・事実確認日・参照元があり、数値・期限は断定しすぎない

「文字数だけ満たす」「雛形マーカーを消しただけ」はプロ水準ではありません。

---

## 文体（誰が読んでもわかりやすく、専門家の信頼を損なわない）

| 方針 | 良い例 | 避ける例 |
|------|--------|----------|
| です・ます調で統一 | 「申込期限は公式要項で確認します」 | 「申込期限は確認される」 |
| 1文は短く（目安 40〜60字、最大 72字） | 長い条件文は2文に分ける | 100字超の一文 |
| 専門語は初出で補足 | 「指定数量（貯蔵できる上限）」 | 略語だけ並べる |
| 誰が・いつ・何を | 「受験者が申込開始日から1週間以内に…」 | 「早めに申し込む」 |
| 法令は要約して説明 | 「35条書面は、売買の場合に…」 | 条文の写しだけ |
| 日常語に言い換え | 「および」→「と」 | 「当該」「前述」「において」 |

`validate_csv.py` / `audit_editorial_quality.py` が検出する主なパターン:

- **ERROR** … `【記入】` `差し替えてください` など未執筆マーカー
- **WARN** … 読みにくい硬語、汎用フレーズの多用、FAQの回答の重複、文が長すぎる

## 試験ガイド（guide_articles.csv）

### 公開ページの役割

検索意図を **1記事1つ** に絞り、制度・手続・学習設計・過去問の使い方など **行動まで落とし込む** 長文記事です。  
**用語の定義・比較表・数値一覧は知識ハブ（`/terms/`）に書き、ガイドからリンクする** — [content-positioning.md](./content-positioning.md)。

### 文字数（`content_status` が `published` の行）

| 列 | ERROR（必須） | 推奨（プロ） |
|----|---------------|--------------|
| `lead` | — | 80字以上 |
| `meta_description` | — | 70〜165字 |
| `user_intent` | — | 50字以上 |
| `section_*_body`（見出しとセット） | **180字以上** | 180〜300字 |
| `faq_*_answer` | **100字以上** | 100〜200字 |
| `action_items` | — | 3項目以上、各10字以上 |
| `related_links`（内部 slug） | — | 2件以上 |

信頼性列（`author_name`, `reviewer_name`, `fact_checked_at`, `primary_sources`）は空だと **WARN**。本番では必ず埋め、`example.com` は実公式 URL に差し替えます。

### 見出し・本文の書き方

- 見出し **5〜7個**。各本文は **空行 `\n\n` で段落分け**（2段落以上推奨）
- 箇条書きにしたいときは本文中で **セミコロン区切り**（`項目1;項目2`）→ ビルドで `<ul>` 化
- 各 section に **具体例・手順・確認先** を1つ以上（数字・日程・画面名・ページ名）
- 最後の section で **関連ガイド・用語・過去問** への次の行動を明示

### FAQ（2件以上必須）

1. 読者の不安に直球で答える質問（？で終える）
2. 回答は **100字以上**。「公式で確認してください」だけで終わらせない
3. 2つの FAQ で **同じ言い回しを繰り返さない**

## 用語詳細（glossary_terms.csv）

### 全用語が詳細記事（ERROR で強制）

**1用語1ページの辞書記事。** 学習計画・申込手順は試験ガイドへ誘導し、本文に長く書かない — [content-positioning.md](./content-positioning.md)。

| 列 | ERROR（必須） |
|----|---------------|
| `article_lead` | 60字以上 |
| `term_detail_body` | **180字以上**、**2段落以上**（`\n\n`） |
| `definition` | 50字以上 |
| `explanation` | 80字以上（選択肢の論点） |
| `common_mistakes` | 40字以上 |
| `memory_tip` | 25字以上 |
| `faq_1`〜`faq_4` 回答 | 各 **100字以上** |
| `exam_points` | 2項目以上（各8字以上） |
| `related_terms` | 登録済み用語2件以上 |

`importance` が **A / S** の用語は、`exam_points` **3項目以上**・`related_terms` **3件以上**を推奨（WARN）。

### FAQ 4件の役割（内容を分ける）

| # | 役割 | 質問の例 |
|---|------|----------|
| 1 | 定義 | 「〇〇とは何ですか？」 |
| 2 | 誤解 | 「よくある誤解は？」 |
| 3 | 試験 | 「試験でどう問われますか？」 |
| 4 | 次の行動 | 「学んだあとに何を確認しますか？」 |

4回答をコピペしない。定義・誤答パターン・出題の型・演習/公式への導線で **役割分担** する。

### 専門家らしさのチェック（人手 + WARN）

- `term_detail_body` または `explanation` に、**数字・期限・条文・「例えば」** など具体性のある表現があるか
- `explanation` に「選択肢」「試験」「正誤」など **出題の型** が書いてあるか
- 演習 CSV の `tags` に `term:用語名` があり、問と用語が双方向につながっているか

## AI・外注への依頼ブリーフ（コピペ用）

```
対象: （試験名）（記事 slug または用語名）
検索意図: （1つだけ）
読者: （例: 初受験・独学・直前）
必須参照: （公式URL）
使ってよい内部リンク: （slug または用語名のリスト）

文体: です・ます、1文60字以内、当該・前述・及びは使わない
禁止: 【記入】マーカー、運用メモ、CSVの話、全記事共通の定型FAQ

試験ガイドの場合:
- section 5〜7本、各180〜300字、段落2つ以上
- FAQ 2組、回答100字以上
- action_items 3項目以上

用語の場合:
- term_detail_body 180字以上・2段落
- FAQ 4組、役割分担（定義/誤解/試験/次の行動）
- related_terms 2件以上（登録済み用語名）
```

## 公開前コマンド

```bash
python3 tools/validate_csv.py
python3 tools/audit_editorial_quality.py
python3 tools/audit_glossary_article_quality.py   # 用語: 演習連動・定型検出
python3 tools/build_all.py
```

本番ゲート（WARN も失敗）:

```bash
python3 tools/audit_editorial_quality.py --strict
```

`build_all.py` は `audit_editorial_quality.py`（ERROR のみ）を含みます。

## リライトの優先順位

1. 未執筆マーカー・80字未満の section（ERROR）を解消
2. 汎用フレーズ・FAQ重複（WARN）を資格固有の文に差し替え
3. 重要用語（A/S）と流入の多いガイド slug から 180字・FAQ100字を満たす
4. `term:` タグ・関連リンクで回遊を足す
