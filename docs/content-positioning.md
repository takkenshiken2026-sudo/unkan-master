# コンテンツ立ち位置：試験ガイド × 知識ハブ（用語解説）

資格サイトの読み物は **「知識を調べる場所」** と **「学習を進める場所」** に分けます。  
検索意図・執筆ルール・内部リンクはこの文書を正本とします。

## 1. 2本柱 + 演習（サイト全体像）

```
┌─────────────────────────────────────────────────────────────┐
│  演習（Do）  /q/ … 過去問・一問一答・模試・復習記録          │
│       ↑ 知識で理解 → ガイドで計画 → 演習で定着              │
├──────────────────────────┬──────────────────────────────────┤
│  知識ハブ（Know） /terms/ │  試験ガイド（Do plan） /articles/ │
│  ・用語解説（1語1ページ）  │  ・制度・手続・学習設計           │
│  ・比較・整理表            │  ・過去問の進め方・直前・再受験   │
│  ・数値・期限早見表        │  ・用語ハブの「使い方」（定義以外）│
│  ・よくある誤答            │                                   │
└──────────────────────────┴──────────────────────────────────┘
```

| 層 | URL | 読者の問い | 記事の単位 | 典型検索 |
|----|-----|------------|------------|----------|
| **知識ハブ** | `/terms/` 配下 | 〇〇**とは**／**違い**／**何日**／**引っかけ** | 1語・1比較・1数値セット・1誤答パターン | `宅建 35条 37条 違い` |
| **試験ガイド** | `/articles/` | **どう**学ぶ／**いつ**申込む／**何から**始める | 1検索意図＝1記事（手順・計画） | `宅建 独学 勉強法` |
| **演習** | `/q/` | **解いて**確認したい | 1問 | `宅建 過去問 2024` |

**原則:** 知識は `/terms/` に集約し、ガイドに用語の定義を書き足さない。逆に用語記事に学習スケジュール全文を書かない。

## 2. 検索意図の振り分け（執筆者向け）

```
読者の頭の中
    │
    ├─ 「この語の意味・試験での論点が知りたい」
    │       → 用語解説（glossary_terms.csv）
    │
    ├─ 「AとBの違い・表で整理したい」
    │       → 比較・整理表（comparisons.csv）
    │
    ├─ 「数字・日数・割合を一覧で見たい」
    │       → 数値・期限早見表（numbers.csv）
    │
    ├─ 「この肢、なぜ間違いか／似た選択肢の罠」
    │       → よくある誤答（mistakes.csv）
    │
    └─ 「勉強の進め方・申込・計画・再受験・直前」
            → 試験ガイド（guide_articles.csv）
```

### 2.1 境界の具体例

| テーマ | 書く場所 | 書かない場所 |
|--------|----------|--------------|
| 35条書面の定義と試験論点 | 用語「35条書面」 | 試験ガイド（リンクのみ） |
| 35条と37条の違い | 比較表 | 用語2ページに重複定義 |
| 建ぺい率の数値一覧 | 数値早見表 | ガイド本文に表丸ごと |
| 用語集の効率的な使い方 | 試験ガイド（genre: **用語ハブ活用法**） | 各用語ページ |
| 3ヶ月学習計画 | 試験ガイド（学習計画） | 用語「学習計画」だけで済ませない |
| 再受験の戦略 | 試験ガイド（注意点・更新） | 用語1件の定義記事 |

### 2.2 重複してよい／ダメなこと

| OK | NG |
|----|-----|
| ガイド本文で用語名を出し、用語ページへリンク | ガイドに用語記事と同じ定義段落をコピペ |
| 用語 FAQ「次に何をする？」で過去問・ガイド slug へ誘導 | 用語記事に週間スケジュール表を載せる |
| 比較表から関連用語2件へリンク | 比較内容を用語本文に全部再掲 |

## 3. 試験ガイド（/articles/）の立ち位置

### 役割

**受験行動の設計図。** 制度の読み方、計画、演習の回し方、直前・再受験までを **時系列・フェーズ** で案内する。

### 12ジャンルとの対応

| フェーズ | 読者状態 | 代表 slug |
|----------|----------|-----------|
| 制度を知る | 受験を検討・初年度確認 | `exam-overview`, `exam-eligibility` |
| 学習を設計する | 教材・期間を決める | `study-plan-*`, `self-study-*` |
| 演習と定着 | 過去問・弱点を回す | `past-questions-*`, `field-*` |
| 直前と当日 | 試験直前 | `final-week-prep`, `exam-day-*` |
| 横断 | 合格後・再受験・改定 | `retake-strategy`, `exam-changes` |

### genre「用語ハブ活用法」（旧: 用語整理）

**用語の定義記事ではない。** 知識ハブ4タブの使い方・回遊・過去問との往復を書くメタ記事専用。

対象 slug 例: `glossary-how-to`, `glossary-study-method`, `confusing-terms`, `numbers-and-deadlines`（**早見表の使い方**として書く。数値の羅列は numbers.csv へ）

## 4. 知識ハブ（/terms/）の立ち位置

### 役割

**試験知識の辞書・資料室。** ブラウジングと検索で「答え」を返す。行動計画は書かない。

### 4タブ

| タブ | データ | 1件の意味 |
|------|--------|-----------|
| 用語解説 | `glossary_terms.csv` | 1用語 = 定義・根拠・試験論点・FAQ4 |
| 比較・整理表 | `comparisons.csv` | 2〜3項目の差分 |
| 数値・期限早見表 | `numbers.csv` | 数字・日数の一覧 |
| よくある誤答 | `mistakes.csv` | 過去問の引っかけ肢パターン |

ナビ上のラベルは **「用語解説」**（入口）だが、サイト内説明では **「知識ハブ」** と呼び、4タブ全体を指す。

## 5. 内部リンク設計

### 5.1 ガイド → 知識

- 本文・FAQ で **初出の専門語** は、可能なら用語名をそのまま書き、公開後に `/terms/g-*.html` へリンク（将来 `related_terms` 相当の列拡張可。現状は本文 HTML または `related_links` は **ガイド slug のみ** のため、section 本文に用語 URL を明記する運用）
- **最低限:** 用語ハブ活用法ジャンルの記事から `terms/index.html` および各タブ index へ必ずリンク

### 5.2 知識 → ガイド

- 用語 FAQ 4（次の行動）で `articles/` の該当 slug を案内（例: 過去問の使い方 → `past-question-strategy`）
- 比較・誤答記事の末尾で「学習計画」「分野別対策」ガイドへ1本

### 5.3 ガイド ↔ ガイド

- `related_links` は同 CSV 内 slug のみ（既存ルール）
- フェーズ順: 概要 → 計画 → 過去問 → 直前

## 6. 執筆チェックリスト（公開前）

**試験ガイド**

- [ ] 検索意図は「進め方・手続・計画」のいずれか1つ
- [ ] 用語の定義を2段落以上書いていない（書いたら用語ページへ移す）
- [ ] `action_items` が演習または知識ハブへの具体行動になっている
- [ ] タイトルに試験名（またはサイト共通略称）が入っている

**用語・知識ハブ**

- [ ] 1語（または1比較・1数値セット）に検索意図が集中
- [ ] 学習スケジュール・申込手順を主題にしていない
- [ ] FAQ4 がガイドまたは演習への次の一歩になっている
- [ ] 強い混同は比較表または誤答タブに切り出したか検討した
