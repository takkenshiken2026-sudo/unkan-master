# 試験ガイドのジャンル（MECE・12区分）

複数資格に展開し **50本以内** の記事を置くとき、記事の `genre` は **`site-config.json` の `guideArticleGenres`** で定義した **12ラベルのみ** を使います。`tools/validate_csv.py` がホワイトリスト検証します。

- **試験ガイドと用語解説の立ち位置:** [content-positioning.md](./content-positioning.md)（正本）
- 記事 slug の具体例: **[guide-article-catalog.md](./guide-article-catalog.md)**
- 新規記事の CSV テンプレ: **[guide-article-template.md](./guide-article-template.md)** / `tools/scaffold_guide_article.py`

## 設計方針

- **12区分で足りる理由**: 記事数はジャンル数ではなく、**同ジャンル内の検索意図の分割**（期間別・分野別・対象者別）で増やす。細かすぎるジャンルは運用と一覧UIが重くなる。
- **相互に排他的（ME）**: 1記事は1ジャンル。迷ったら読者の主目的で決める（例: 申込期限→受験・申込、合格後手続き→注意点・更新）。
- **資格非依存**: ラベルに試験名を入れない。

## フェーズとジャンル一覧（12）

| フェーズ | genre | 主な検索意図 | 50本以内で置く軸 |
|----------|-------|--------------|-----------------|
| 制度を知る | 試験概要 | 全体像、公式の見方、サイトの使い方 | 公式情報・比較・初めて |
| 制度を知る | 受験・申込 | 資格、日程、手数料、申込、免除 | 手続き別・地域別 |
| 制度を知る | 合格・難易度 | 合格率、合格点、難易度 | 年度別・初学者向け |
| 制度を知る | 出題・形式 | 出題範囲、科目、形式、CBT | 改定・科目別 |
| 学習を設計する | 学習計画 | 期間、週間表、働きながら | 3ヶ月/6ヶ月/短期 |
| 学習を設計する | 独学対策 | 独学、教材、継続 | 教材別・つまずき別（**アフィリエイト記事10本目安**は主にここ。`tags` に `アフィリエイト`） |
| 演習と定着 | 過去問活用 | 過去問、模試、解き直し | 年度別・弱点別 |
| 演習と定着 | 分野別対策 | 分野ごとの基礎・演習 | **fields ごとに複製** |
| 演習と定着 | 用語ハブ活用法 | **知識ハブ4タブの使い方**（用語定義は書かない） | 混同語・数字の整理**の進め方** |
| 演習と定着 | 復習・苦手克服 | 復習サイクル、苦手 | ノート・間隔 |
| 直前と当日 | 直前・当日 | 直前1〜2週間、当日 | 持ち物・会場 |
| 横断 | 注意点・更新 | 誤解、改定、合格後、再受験 | 制度変更・FAQ |

## 21区分から統合したもの（参考）

| 旧 genre（統合前） | 新 genre |
|-------------------|----------|
| 受験資格 / 日程・申込 | 受験・申込 |
| 試験形式 / 出題範囲 | 出題・形式 |
| 教材選び | 独学対策（記事タイトルで教材を明示） |
| 模試・演習 | 過去問活用 |
| 数値・計算 | 用語ハブ活用法 または 分野別対策 |
| 直前対策 / 試験当日 | 直前・当日 |
| 合格後手続き / 再受験 / 制度変更 / よくある誤解 | 注意点・更新 |

## 表示スタイル（`style`）

| style | ジャンル |
|-------|----------|
| overview | 試験概要 |
| institution | 受験・申込、合格・難易度、出題・形式 |
| study | 学習計画、独学対策 |
| practice | 過去問活用、分野別対策、用語ハブ活用法、復習・苦手克服 |
| final | 直前・当日 |
| meta | 注意点・更新 |
